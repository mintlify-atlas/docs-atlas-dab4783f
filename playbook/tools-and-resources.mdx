Choosing the right tools shapes how effectively you can work with AI. This guide organizes the Vibe Coding ecosystem into categories — IDEs and terminals, AI models, development utilities, prompt and skill resources, and communities — so you can find the right tool for each part of your workflow. Model performance tiers are included to help you match model capability to task complexity.

## IDEs and terminals

<CardGroup cols={2}>
  <Card title="Visual Studio Code" icon="code" href="https://code.visualstudio.com/">
    A powerful IDE suited for code reading and manual editing. The **Local History** plugin is especially useful for lightweight version tracking within a session.
  </Card>
  <Card title="Cursor" icon="arrow-pointer" href="https://cursor.com/">
    An AI-native IDE built on VS Code. Well-established in the developer community with integrated AI editing and chat.
  </Card>
  <Card title="Warp" icon="terminal" href="https://www.warp.dev/">
    A modern terminal with built-in AI assistance. Improves command-line operations and error diagnosis without leaving the terminal.
  </Card>
  <Card title="Neovim + LazyVim" icon="keyboard" href="https://github.com/neovim/neovim">
    High-performance modal editor with a rich plugin ecosystem. [LazyVim](https://github.com/LazyVim/LazyVim) provides a pre-configured setup with LSP, completion, and debugging out of the box.
  </Card>
</CardGroup>

<Tip>
Run Claude Code or Codex CLI inside the VS Code integrated terminal. You get full diff views and context access without leaving your workspace.
</Tip>

## AI models

The following models are actively used in the Vibe Coding community. Each has different strengths, access methods, and cost profiles.

<CardGroup cols={2}>
  <Card title="Claude Opus 4.6" icon="asterisk" href="https://claude.ai/new">
    The primary model for complex Vibe Coding work. Available via Claude Code CLI, IDE plugins, and the web interface. Strong at multi-file reasoning, long-context tasks, and following structured instructions.
  </Card>
  <Card title="GPT Codex (xhigh)" icon="brain" href="https://chatgpt.com/codex/">
    Well-suited for large projects and complex logic. Available via Codex CLI. Use `--yolo` flag to skip confirmation prompts in automated workflows.
  </Card>
  <Card title="Gemini CLI" icon="google" href="https://geminicli.com/">
    Free access to Gemini models. Good for running scripts, organizing documentation, and exploring ideas. Also available through [AI Studio](https://aistudio.google.com/prompts/new_chat) and [Gemini Enterprise](https://cloud.google.com/gemini-enterprise).
  </Card>
  <Card title="Kiro" icon="tool" href="https://kiro.dev/">
    Provides free access to Claude Opus 4.6 with a client and CLI tool. A useful option when managing subscription costs.
  </Card>
  <Card title="GLM" icon="flag" href="https://bigmodel.cn/">
    Large language model from Zhipu AI. Supported natively in OpenCode CLI configuration for free-tier usage.
  </Card>
  <Card title="Qwen" icon="flag" href="https://qwenlm.github.io/qwen-code-docs/zh/cli/">
    Alibaba's AI model with a CLI tool that includes a free usage quota. Suitable for general coding tasks.
  </Card>
  <Card title="Kimi K2.5" icon="globe" href="https://www.kimi.com/">
    A Chinese-developed AI model suitable for general tasks. Good alternative when other models are unavailable.
  </Card>
  <Card title="Droid" icon="terminal" href="https://factory.ai/news/terminal-bench">
    Provides CLI access to Claude Opus 4.6 and other models. Designed for terminal-first workflows.
  </Card>
</CardGroup>

## Model performance tiers

Use this reference to match model capability to task complexity. Not all tasks need the highest-tier model — using a lighter model for simple work preserves quota for the tasks that matter.

<AccordionGroup>
  <Accordion title="Top tier: complex tasks">
    **Recommended: gpt-5.2-xhigh (Codex)**

    Use the top-tier model for:
    - Multi-file architectural changes
    - Debugging complex, cross-module issues
    - Generating implementation plans from scratch
    - Tasks where a single wrong output costs significant time to fix

    The community recommendation is to use top-tier models exclusively for complex tasks to ensure the best results and efficiency.
  </Accordion>
  <Accordion title="Mid tier: standard development">
    **Claude Opus 4.6, Gemini 3.0 Pro**

    Use mid-tier models for:
    - Single-file feature implementation
    - Writing or reviewing unit tests
    - Code explanation and documentation
    - Most day-to-day Vibe Coding steps
  </Accordion>
  <Accordion title="Light tier: small tasks">
    **GPT Codex (medium), Gemini Nano**

    Use light models for:
    - Small fixes and single-function changes
    - Formatting and renaming
    - Simple script generation
    - Tasks you could verify in under a minute
  </Accordion>
  <Accordion title="Specialist models">
    | Task | Model |
    |---|---|
    | Marketing copy | Opus 4.1 |
    | 2D sprite generation | ChatGPT + Nano Banana |
    | Music generation | Suno |
    | Sound effects | ElevenLabs |
    | Video generation | Sora 2 |
  </Accordion>
</AccordionGroup>

## Virtual environment and Python tooling

For Python projects, always use a virtual environment. It isolates dependencies per project and makes environment setup reproducible:

```bash
# Create a virtual environment
python3 -m venv .venv

# Activate it
source .venv/bin/activate  # macOS/Linux
.venv\Scripts\activate     # Windows

# Install dependencies
pip install -r requirements.txt
```

<Note>
VS Code automatically detects `.venv` and activates it in new terminals. Add `.venv/` to your `.gitignore` so the environment is never committed to the repository.
</Note>
