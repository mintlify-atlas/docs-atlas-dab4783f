AI Skills are higher-level capability modules that go beyond simple prompts. Each skill bundles a trigger definition, activation conditions, boundary rules, deliverables, and reproducible workflow steps into a single `SKILL.md` entrypoint — giving any AI agent expert-level behavior in a specific domain on demand.

## How skills work

<Steps>
  <Step title="Load the skill into your agent context">
    Copy the contents of the relevant `SKILL.md` into your AI assistant's system prompt, project context, or memory file (for example, Claude's `CLAUDE.md` or OpenCode's context window).
  </Step>
  <Step title="State the trigger condition">
    Describe what you need using the keywords and task framing that match the skill's "When to Use" section. The skill's frontmatter `description` is designed to activate automatically in AI tools that support skills natively.
  </Step>
  <Step title="Provide required inputs">
    Each skill lists mandatory inputs. If any are missing, the skill instructs the AI to ask 1–3 targeted questions before proceeding — never making up information.
  </Step>
  <Step title="Receive structured deliverables">
    The skill produces a defined output: a SKILL.md file, an SOP document, an architecture whiteboard JSON, a code review, or a translated document, depending on which skill you loaded.
  </Step>
</Steps>

<Note>
  Skills are not prompts — they are operator manuals. Each one defines what is in scope, what is explicitly out of scope, and includes at least three reproducible examples with acceptance criteria.
</Note>

## Create a custom skill

You can generate a new skill for any domain using the meta-skill. The process takes three steps:

<Steps>
  <Step title="Prepare your domain material">
    Collect the documentation, API specs, code examples, and known failure modes for the domain you want to cover. The more concrete the material, the better the generated skill.
  </Step>
  <Step title="Load skills-skills and provide your material">
    Give the AI both `assets/skills/skills-skills/SKILL.md` and your domain material. The meta-skill will scaffold the directory layout, write the frontmatter, extract patterns, add examples, and apply the quality gate.
  </Step>
  <Step title="Validate and iterate">
    Run the built-in validator: `./assets/skills/skills-skills/scripts/validate-skill.sh assets/skills/<your-skill> --strict`. Fix any warnings before shipping.
  </Step>
</Steps>

<Tip>
  If your source material is a documentation website or GitHub repository, use the bundled Skill Seekers tool to generate a first draft automatically before refining it with the meta-skill.
</Tip>
