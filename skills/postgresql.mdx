## When to trigger this skill

The skill description reads:

> PostgreSQL database documentation — SQL queries, database design, administration, performance tuning, and advanced features. Use when working with PostgreSQL databases, writing SQL, or managing database systems.

Trigger it when you are:

- Writing or debugging SQL queries against a PostgreSQL database
- Designing or migrating a database schema
- Configuring database connections in application code
- Tuning queries or indexes for performance
- Working with PostgreSQL-specific features such as `RETURNING`, predefined roles, `LISTEN`/`NOTIFY`, or row-level security
- Administering a PostgreSQL server (users, roles, permissions)

## Practical examples

### Connecting from Python

```python
import psycopg2

# Keyword/value connection string
conn = psycopg2.connect(
    "host=localhost port=5432 dbname=mydb user=app connect_timeout=10"
)

# URI format — equivalent to the above
conn = psycopg2.connect(
    "postgresql://app@localhost:5432/mydb?connect_timeout=10"
)
```

<Note>
  When untrusted users have access to the database, begin each session by removing publicly-writable schemas from `search_path`. Set `options=-csearch_path=` in the connection string or run `SELECT pg_catalog.set_config('search_path', '', false)` immediately after connecting.
</Note>

### Returning data from a write

```sql
-- Retrieve the generated ID after insert
INSERT INTO users (firstname, lastname)
VALUES ('Ada', 'Lovelace')
RETURNING id;

-- See prices after a bulk update
UPDATE products
SET price = price * 1.10
WHERE price <= 99.99
RETURNING name, price AS new_price;

-- Audit deleted rows without a second query
DELETE FROM products
WHERE obsoletion_date = 'today'
RETURNING *;
```

### Granting predefined roles

```sql
-- Let an admin user send signals to backends
GRANT pg_signal_backend TO admin_user;

-- Allow a monitoring role to read all statistics
GRANT pg_monitor TO monitoring_user;

-- Let a maintenance role run VACUUM and REINDEX on all tables
GRANT pg_maintain TO dba_user;
```

<Warning>
  Predefined roles grant access to privileged capabilities and sensitive system information. Only grant them where genuinely needed, and monitor release notes because specific permissions may change across PostgreSQL versions.
</Warning>

### Multi-host connection string

```text
postgresql://host1:5432,host2:5432/mydb?target_session_attrs=any&application_name=myapp
```

This instructs libpq to try each host in turn and connect to the first that accepts a session matching the `target_session_attrs` policy. Useful for read replicas or high-availability setups.
