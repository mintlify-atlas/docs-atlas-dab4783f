A Vibe Coding environment needs four things: a reliable terminal, version control, a code editor, and at least one AI CLI tool you can invoke from the command line. This guide gives you a prompt-driven path to set all of that up, plus post-installation shortcuts that make switching between AI tools instant.

<Note>
  Complete [network setup](/getting-started/network-setup) before starting here. All package downloads require a working connection to GitHub and npm.
</Note>

## Post-installation: CLI shortcuts

Once your tools are installed, add these aliases to `~/.bashrc` (or `~/.zshrc` on macOS) so you can launch any AI tool with a single letter.

```bash
# c — Codex (GPT, high-reasoning, no confirmation prompts)
alias c='codex --enable web_search_request -m gpt-5.3-codex-max -c model_reasoning_effort="high" --dangerously-bypass-approvals-and-sandbox'

# cc — Claude Code (all permissions granted)
alias cc='claude --dangerously-skip-permissions'

# g — Gemini CLI (YOLO mode, no confirmation prompts)
alias g='gemini --yolo'
```

Apply the changes immediately:

```bash
source ~/.bashrc
```

<Warning>
  The `--dangerously-bypass-approvals-and-sandbox` and `--dangerously-skip-permissions` flags grant the AI full permission to read, write, and execute without asking. Only use these in isolated project directories, never in your home directory or on a shared machine.
</Warning>

## Verify your setup

Run the following to confirm each tool is installed and on your PATH:

```bash
node --version    # v20.x.x or later
git --version     # git version 2.x.x
python3 --version # Python 3.x.x
tmux -V           # tmux 3.x
code --version    # VS Code version (if installed)
gemini --version  # or: opencode --version
```
