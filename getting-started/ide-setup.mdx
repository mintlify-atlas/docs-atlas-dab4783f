Your editor is where you spend most of your time reviewing and adjusting AI-generated code. A well-configured VS Code reduces friction at every step: auto-save keeps the AI's file reads current, Local History gives you a safety net when the AI overwrites something important, and GitLens surfaces the full change history without leaving the editor. This guide covers the setup prompts and key extensions for every platform.

<Note>
  Complete [development environment setup](/getting-started/dev-environment) before this step. You need VS Code installed and, on Windows, WSL2 running.
</Note>

## Recommended extensions

These four extensions are referenced in every setup prompt above. Install them first.

| Extension | Why it matters for Vibe Coding |
|:----------|:-------------------------------|
| **Local History** | Saves a local snapshot every time you save a file. Essential safety net when the AI overwrites code you wanted to keep. |
| **GitLens** | Inline blame, branch history, and commit comparisons — lets you review everything the AI changed in one place. |
| **Prettier** | Consistent code formatting on save. Keeps AI-generated code readable without manual cleanup. |
| **ESLint** | Highlights syntax and style errors inline. Catches issues before you run the code. |

<Tip>
  Install **Local History** before you start any AI-assisted session. AI tools can make sweeping file changes without explicit confirmation. Local History lets you recover any version saved in the last few days with two clicks.
</Tip>

## Cursor setup

Cursor is a VS Code fork with AI editing built in at the editor level — no extension required.

```text
You are a patient Cursor IDE setup assistant. I want to use Cursor as my primary development tool. Guide me step by step through installation and configuration.

My operating system: [Windows / macOS / Linux]

Please guide me in this order, one step at a time. Wait for me to confirm success before moving on:

1. Download and install Cursor (https://cursor.com)
2. Initial launch configuration (sign in, choose theme)
3. Import VS Code settings and extensions (if I used VS Code before)
4. Configure AI features (API key or subscription)
5. Learn core shortcuts:
   - Cmd/Ctrl + K: AI inline edit
   - Cmd/Ctrl + L: AI chat panel
   - Cmd/Ctrl + I: Composer mode (multi-file edit)
6. Configure auto-save
7. Verify AI features work

Requirements:
- Give me the exact operation for each step
- Explain what makes Cursor different from VS Code
- If I encounter a problem, help me diagnose and fix it
- Ask whether each step succeeded before continuing

Start by asking which operating system I use.
```
