Vibe Coding CN is the ultimate workstation for AI pair programming, designed to help developers smoothly turn ideas into reality. The term **Vibe Coding** (氛围编程) was coined by computer scientist [Andrej Karpathy](https://x.com/karpathy) to describe a natural language-driven, LLM-first development style that prioritizes immersive, rapid prototyping. This project extends that concept into a structured, repeatable methodology — one built around planning-driven development, modularity, and index building — so that AI assistance stays reliable and in control, no matter how complex the project gets.

## What Vibe Coding CN provides

The project is organized around four core assets:

- **Prompts library** — hundreds of copy-ready prompts for every stage of development: requirements, planning, execution, debugging, and review
- **Skills** — composable, reusable AI skill modules (each a self-contained `SKILL.md`) for specialized domains such as databases, CLI tools, and multi-agent coordination
- **Workflows** — structured patterns for the full development loop: requirement → context document → implementation plan → step-by-step execution → self-test → progress record
- **Methodology** — first-principles thinking on how to work with AI effectively, including context management, modular architecture, and iterative feedback

## Who this is for

Vibe Coding CN is built for anyone who wants to work with AI coding assistants more effectively:

- **Complete beginners** who have never written code and want AI to guide them through the entire process
- **Developers** who want structured patterns for AI-assisted feature development, refactoring, and debugging
- **Teams** looking to standardize how they use AI tools across projects

<Note>
  This project is continuously evolving. All methods and experiences may become obsolete as AI capabilities advance. Maintain an AI-first mindset and treat every technique as provisional.
</Note>

## From Vibe Coding to Glue Coding

The evolved form of Vibe Coding is **Glue Coding** — a methodology that shifts attention away from writing new code toward composing proven, battle-tested libraries. The principle: copy before writing, connect before creating, reuse before inventing. This approach directly addresses the three critical risks of pure Vibe Coding:

| Problem | Solution |
|---|---|
| AI hallucination | Only use verified, mature code — zero hallucination risk |
| Complexity explosion | Every module is a well-tested wheel |
| High barrier to entry | You only need to describe the connection method |

Learn more in [Glue Coding](/principles/glue-coding).
